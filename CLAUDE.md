## 一、项目概述

| 项目 | 说明 |
|------|------|
| **项目名称** | 青柠记账 |
| **项目定位** | 个人桌面记账工具，帮助用户轻松记录和管理日常花销 |
| **目标平台** | Windows 10+ / macOS 11+ |
| **货币单位** | 人民币（¥） |
| **核心用户** | 不懂编程技术的普通用户（本项目由 Claude Code 辅助开发） |

### 核心功能

1. **花销记录**：记录每一笔支出的金额、日期、分类、备注
2. **二级分类**：一级大类 + 二级小类的双层分类体系
3. **历史查询**：按日期、分类等维度查看过往记账记录
4. **数据统计**：月度/年度消费汇总，分类占比分析
5. **数据导出**：支持导出为 Excel/CSV 格式（后续版本）

## 三、技术栈方案（待选择）

> **选择方式**：请从以下 4 个方案中选择一个。如果你不确定，推荐方案 A（最稳妥）或方案 C（未来可扩展手机端）。

---

### 方案 A：Electron + React + SQLite ⭐ 推荐

| 维度 | 评价 |
|------|------|
| 安装包体积 | 80–150 MB（较大） |
| 内存占用 | 200–400 MB（较高） |
| 启动速度 | 2–5 秒（首次较慢） |
| 开发语言 | JavaScript / TypeScript |
| AI 辅助友好度 | ⭐⭐⭐⭐⭐ 最佳 |
| 跨平台一致性 | ⭐⭐⭐⭐⭐ 完美一致 |
| 打包分发 | ⭐⭐⭐⭐⭐ 最成熟 |

**优势**：
- 技术最成熟，全球使用最广的桌面应用方案（VS Code、Discord、Notion 都用它）
- JavaScript/React 是 AI 训练数据最多的语言，Claude Code 辅助开发效果最佳
- 打包分发一条命令搞定，支持自动更新
- 网上中文教程和解决方案最多，遇到问题容易搜到答案

**劣势**：
- 应用体积大（内置整个浏览器），对老旧电脑不太友好
- 内存占用偏高

**一句话**：最稳妥、最省心的选择，代价是应用比较"重"。

---

### 方案 B：Tauri + Vue + SQLite

| 维度 | 评价 |
|------|------|
| 安装包体积 | 3–10 MB（极小） |
| 内存占用 | 30–50 MB（极低） |
| 启动速度 | <0.2 秒（极快） |
| 开发语言 | JavaScript（前端）+ Rust（后端） |
| AI 辅助友好度 | ⭐⭐⭐ 一般 |
| 跨平台一致性 | ⭐⭐⭐⭐ 良好 |
| 打包分发 | ⭐⭐⭐⭐ 良好 |

**优势**：
- 应用体积极小，启动极快，资源占用几乎可忽略
- 前端用 Vue，上手简单
- 安装包比一张照片还小

**劣势**：
- **后端必须用 Rust 编写**，Rust 学习曲线非常陡峭，对初学者极不友好
- AI 生成 Rust 代码的质量不如 JavaScript 稳定
- 框架较新，API 仍在快速变化中

**一句话**：性能最优美，但 Rust 门槛是硬伤，不推荐零基础用户选择。

---

### 方案 C：Flutter Desktop + SQLite ⭐ 备选推荐

| 维度 | 评价 |
|------|------|
| 安装包体积 | 40–80 MB（中等） |
| 内存占用 | 100–200 MB（中等） |
| 启动速度 | ~1 秒（快） |
| 开发语言 | Dart |
| AI 辅助友好度 | ⭐⭐⭐⭐ 较高 |
| 跨平台一致性 | ⭐⭐⭐⭐⭐ 像素级一致 |
| 打包分发 | ⭐⭐⭐ 中等 |

**优势**：
- Google 官方维护，社区活跃度最高（GitHub 176k+ Stars）
- UI 渲染流畅（60fps），体验接近原生应用
- **最大亮点**：同一套代码可直接打包为 iOS 和 Android 手机应用，未来扩展手机端几乎零成本
- 记账软件天然适合手机端（随时随地记录花销），这个优势非常有价值

**劣势**：
- 需要学习 Dart 语言（但难度不高，和 JavaScript 相当）
- Mac 端打包签名配置稍复杂
- Windows 端需要额外工具制作安装包

**一句话**：现在做桌面版，未来免费得手机版，长远来看可能是最正确的选择。

---

### 方案 D：Python + PySide6 + SQLite

| 维度 | 评价 |
|------|------|
| 安装包体积 | 40–120 MB（中等） |
| 内存占用 | 50–85 MB（较低） |
| 启动速度 | 1–2.5 秒（中等） |
| 开发语言 | Python |
| AI 辅助友好度 | ⭐⭐⭐⭐⭐ 极佳 |
| 跨平台一致性 | ⭐⭐⭐ 一般 |
| 打包分发 | ⭐⭐ 较差 |

**优势**：
- Python 语法最简单，零基础入门最快
- SQLite 是 Python 自带模块，无需安装任何额外东西
- AI 生成 Python 代码质量极高

**劣势**：
- **打包分发是最大的坑**：需要安装 C 编译器，配置复杂，容易出错
- 在 Windows 上无法打包 Mac 版本（反之亦然），需要分别在两台电脑上操作
- UI 布局代码冗长，界面美观度不如 Web 技术方案
- 不支持内置自动更新

**一句话**：开发阶段最轻松，但打包分发环节的痛苦会抵消前面的轻松。

## 四、项目开发规则

### 🔴 核心规则：技术决策必须由用户拍板

> **用户不懂编程技术，在整个项目开发过程中，任何涉及技术选择的事项（包括但不限于：技术栈选型、库/框架选择、架构方案、UI 设计风格、功能实现方式等），Claude 必须：**
>
> 1. **列出至少 2–3 个可行方案**
> 2. **用通俗易懂的语言向用户解释每个方案的优劣势**
> 3. **给出自己的推荐意见和理由**
> 4. **等待用户明确做出选择后，再继续执行**
>
> **严禁自行替用户做技术决策，严禁跳过方案对比环节直接实施。**

### 其他开发规则

1. **中文优先**：项目全程使用中文 — 代码注释、Git 提交信息、UI 界面、文档说明全部使用中文（用户看不懂英文）
2. **代码注释**：所有关键逻辑需添加中文注释，方便用户理解
3. **提交规范**：Git 提交信息使用中文，清晰描述每次改动
3. **文档同步**：CLAUDE.md 是项目的核心文档，重大变更需同步更新
4. **渐进开发**：优先实现核心功能（记账、分类、查询），再逐步增加高级功能（统计、导出、图表）
5. **UI 简洁**：界面设计以简洁直观为原则，避免过度复杂的交互
